# hoi4_mnds_cursor_changer
Hearts of Iron IV (HoI4) のマウスポインタ（.cur および .ani）を、ローカルファイルの物理的な上書きによって「もんど☆すとらの」の二次創作カーソルへの変更とデータの管理をするPythonスクリプトです。

* Steam側で「ファイルの整合性を確認」を実行すると、上書きされたカスタムカーソルはすべて公式の状態（バニラ）にリセットされます。

## 💡 開発背景
Clausewitzエンジン（HoI4）におけるマウスカーソルの制御ロジックは、コアシステム側にハードコードされています。
通常のMod構成（replace_path や同名ファイルの上書き配置）を用いた場合でも、Mod側から上書き処理をすることは困難です。

本ツールは、このエンジンの制約を外部スクリプトによる物理パッチという力技で突破し、アニメーションカーソルを含めた陣営ごとの完全なUIカスタマイズを実現するために作成されました。

## ✨ 主な機能
1. クロスプラットフォーム対応
   Windows, macOS(検証中-つまりうまくいかない), Linux(未検証) を実行時に自動検知し、各OSのSteamデフォルトインストールパスを自動で発見します。
2. 差分適用時の混ざり（クロスコンタミネーション）防止
   陣営ごとに用意されているカスタムカーソルの数が異なる場合（一部のファイルしか用意していない差分状態など）を想定しています。新しい陣営を適用する際、裏側で一度バニラ状態に完全リセットしてから上書きを行うため、前回の陣営のカーソルが残留するバグを防ぎます。
3. 安全なバックアップ＆クリーンな現状復帰
   初回実行時に gfx/cursors のオリジナルデータを cursors_backup として安全に退避させます。
   メニューから「バニラ環境に復元」を実行した際は、ファイルを元に戻します。これにより、将来的なゲーム本体のアップデート時に古いバックアップデータがシステムと干渉するリスクを排除します。

## 🚀 使用方法 (Usage)

### 前提条件
* Pythonの実行環境が必要です。
    * わからない人は、AIに「Pythonのインストール方法を教えて」と聞いてみてください。最近のAIはとても賢いので、丁寧に教えてくれると思います。
    

### 実行手順
1. 本スクリプトと custom_cursors フォルダ一式を任意の作業ディレクトリに配置します。
2. ターミナル（コマンドプロンプト / PowerShell / Macのターミナル）を開き、スクリプトのある階層へ移動します。
    * cd コマンドで移動するか、ターミナルにフォルダをドラッグ＆ドロップしてください。
3. 以下のコマンドを実行します。
   python cursor_patcher.py
4. CLI上にインタラクティブメニューが表示されます。適用したい操作の番号（1 または 2）を入力してください。
5. パッチ完了のログを確認後、通常通りHoI4を起動してください。

### アンインストール（バニラへの復元）
スクリプトを再度実行し、メニューから [3] バニラ (Vanilla) 環境に復元してクリーンアップ を選択します。
バニラファイルが完全復元されます。

## 🛠 トラブルシューティング
* 「デフォルトのパスにHoI4が見つかりませんでした」と表示される場合
  Steamライブラリをデフォルト（Cドライブなど）以外のカスタムドライブに設定している場合、自動検出をフォールバックします。その場合は、プロンプトの指示に従って Hearts of Iron IV のインストール先ルートディレクトリの絶対パスを手動で入力（またはターミナルにドラッグ＆ドロップ）してください。
